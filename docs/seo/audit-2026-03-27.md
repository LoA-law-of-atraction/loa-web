## Critical Issues

### 1. All public pages are `"use client"` — Google sees empty HTML
**Categories:** Technical SEO, On-Page SEO, Core Web Vitals

`app/page.js`, `app/features/page.js`, `app/about-us/page.js`, `app/pricing/page.js`, `app/download/page.js`, and `app/updates/page.js` all have `"use client"` at line 1. In Next.js App Router this means **zero server-rendered HTML** — Googlebot receives an empty shell. All text, H1, meta tags, and content require JavaScript to appear. This delays indexation by days to weeks and kills LCP.

The fix pattern already exists in `app/privacy-policy/page.js` — a server wrapper exports `metadata`, a `"use client"` child handles interactivity. Apply it to every public page.

```js
// app/features/page.js (server component — no "use client")
export const metadata = { title: "Features | LoA", description: "..." };
export default function FeaturesPage() {
  return <FeaturesClient />;
}
```

**Files:**
- `app/page.js:1`
- `app/features/page.js:1`
- `app/about-us/page.js:1`
- `app/pricing/page.js:1`
- `app/download/page.js:1`
- `app/updates/page.js:1`

---

### 2. `StructuredData.js` points to staging domain
**File:** `components/StructuredData.js:12-13`

Both `Organization.url` and `Organization.logo` are set to `https://loa-web-landing.vercel.app`. Every structured data block Google parses is attributed to the wrong domain — this actively prevents Knowledge Panel eligibility and entity association with the production site. One-line fix with outsized impact.

**Fix:** Change both values to `https://www.loa-lawofattraction.co`.

---

### 3. Hardcoded fabricated `aggregateRating`
**File:** `components/StructuredData.js:33`

`ratingValue: "4.8"` and `ratingCount: "1000"` are static strings with no connection to real App Store or Google Play reviews. Under the September 2025 QRG this is grounds for a manual quality action. Remove until real verified counts are available, or pull dynamically from the store APIs.

---

### 4. No named founders or authors anywhere on site
**Category:** E-E-A-T Trustworthiness

The About page has no team section, no founder photos, no LinkedIn links. For a wellness app requesting system-level phone permissions and charging subscription fees, anonymity is a critical trust gap. This is the single highest-leverage E-E-A-T improvement available.

**File:** `app/about-us/page.js`

---

### 5. LCP estimated 3.5–5s+ (FAIL)
**Files:** `app/page.js:1`, `components/SolarSystemBackground.js`

Three compounding causes:
- `"use client"` on homepage forces full JS execution before any paint
- 60fps canvas with `shadowBlur` on 8 planets runs on the main thread from load
- H1 has `transition={{ delay: 0.3, duration: 0.8 }}` — paint delayed 300ms after JS hydration

**Fix:** Remove `"use client"` from `page.js`. Wrap `SolarSystemBackground` in `dynamic(() => import(...), { ssr: false })`.

## Top 5 Actions by ROI

1. **Remove `"use client"` from all public page files** — fixes JS rendering, unlocks metadata, unblocks LCP. Use the server wrapper + client component pattern from `app/privacy-policy/page.js`.
2. **Fix `StructuredData.js` staging URL** — one-line change in `components/StructuredData.js:12-13`, fixes entity association for all structured data Google has indexed.
3. **Add founder/team identities to About page** — highest single E-E-A-T lift; no code complexity.
4. **Wrap `SolarSystemBackground` in `dynamic(..., { ssr: false })`** — removes canvas from critical render path, significantly drops LCP.
5. **Replace Google Fonts `@import` with `next/font/google`** — eliminates render-blocking font request chain.
